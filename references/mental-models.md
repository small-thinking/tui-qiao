# Critical Thinking & ROI Models

## 1. First-Principles Deconstruction
When evaluating a community trend or message, strip it of its "vibes" and analogies:
- **Core Truths:** What are the non-negotiable facts?
- **Assumptions:** What is being taken for granted? (e.g., "More tokens = more intelligence")
- **Alternative Paths:** How else can we achieve the same goal without the hype?

## 2. ROI over Vibes (The "Post-Hype" Check)
Specifically designed for trends like "Token Flex" or "Hardware FOMO":
- **Input Cost:** Tokens, subscription fees, hardware electricity, human time.
- **Output Value:** Did the task result in a deployable feature, a verified research finding, or just a "chat experience"?
- **Sanity Question:** Would you pay for this out of your own pocket if you didn't have a corporate budget?

## 3. Inversion (Thinking Backwards)
Instead of asking "How do I use the best tool?", ask:
- "What happens if I use the leanest tool (Gemini-CLI/Codex)?"
- "What failure state are we trying to avoid by buying this new hardware?"

## 4. Occam's Razor for AI
- If a task can be solved by a $0.01 prompt on Gemini, using a $5.00 reasoning chain on Claude is a technical debt, not a "flex".
- Complexity is a cost, not a feature.
